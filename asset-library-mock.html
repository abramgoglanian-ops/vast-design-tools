Asset Library — two structure options

One top-level Asset Library for everything approved. Same idea, two ways to arrange it.

The shared idea. The Asset Library has one job: the single place you go for anything approved — logos, fonts, color, icons, illustration, key art, renders, photos, video, motion. People browse it type-first (Hillary's order: asset type, then product, then system). Keana's reorg stays exactly as she built it — product-first (product, then system, then type, then selects) — and serves as the source of truth underneath. Alias links bridge the two, so the type-first view is just pointers into Keana's product-first selects. One real file, no duplication, and it updates the moment she updates a select.

approved brand assets (real files) alias → points into the source Keana's product-first source of truth raw / originals

The two options

OPTION A

Two rooms

Both mental models are first-class at the top of the Asset Library: an approved browse-by-type room, and the full library by product.
Asset Library/ ├─ Approved — browse by type/ type-first │ ├─ Logos & Identity/ real files │ ├─ Typography/ Color/ Icons/ │ ├─ Illustration/ Key Art/ │ ├─ Photography/ │ │ ├─ Haven-1 ↗ alias │ │ ├─ Haven Station ↗ alias │ │ └─ Astronauts ↗ alias │ ├─ Video/ ↗ aliases │ ├─ Renders & CGI/ ↗ aliases │ └─ Motion/ ↗ aliases └─ Full Library — by product/ source ├─ HAVEN STATION/ │ └─ PRIMARY STRUCTURE/ │ └─ PHOTO · VIDEO · 3D │ └─ SELECTS/ ← aliases hit here ├─ SATELLITES/ ASTRONAUTS/ … └─ __ORIGINAL ASSETS/ raw

Trade-offs

Both audiences served at the top: browse-by-type for most people, by-product for media pros.
Clear line between the curated approved set and the full library.
Two things at the top of the folder to explain.
The full library being right there can tempt people to grab raw instead of approved.
OPTION B

Front door

Type-first is the entire face of the Asset Library. Keana's product-first structure sits one level down as the source.
Asset Library/ type-first is the whole face ├─ Logos & Identity/ real files ├─ Typography/ Color/ ├─ Iconography/ Illustration/ ├─ Key Art/ ├─ Photography/ │ ├─ Haven-1 ↗ alias │ ├─ Haven Station ↗ alias │ └─ Astronauts ↗ alias ├─ Video/ ↗ aliases ├─ Renders & CGI/ ↗ aliases ├─ Motion/ ↗ aliases └─ _Source (by product)/ tucked down ├─ HAVEN STATION/ │ └─ PRIMARY STRUCTURE/ │ └─ PHOTO · VIDEO · 3D │ └─ SELECTS/ ├─ SATELLITES/ ASTRONAUTS/ … └─ __ORIGINAL ASSETS/

Trade-offs

Cleanest face. Type-first IS the structure, matching Hillary's model exactly.
Steers everyone to the approved, type-first view first.
Media pros go one level down (_Source) to reach the product-first working structure.
Keana's source is de-emphasized — worth confirming she's good with it being tucked.

How the alias bridge works

An entry in the type-first view is a pointer, not a copy. Following it lands you in Keana's product-first selects:

Asset Library / Photography / Haven Station ↗ points to …/HAVEN STATION / PRIMARY STRUCTURE / PHOTO / SELECTS

Because it's a pointer, there's one real file. Keana keeps working product-first; everyone else browses type-first; both see the same asset.

Anticipated friction with aliases (and how we take it off the team)

Nobody creates aliases today
Most people never will. Almost everyone only ever opens an alias, which behaves exactly like opening a folder — no new skill needed.
Who actually makes them
A small curator group, not the whole team: Keana for photo/video selects, you and Jessie for renders, key art, and brand assets.
Day-one setup feels huge
The initial product aliases get generated in bulk by script (the shortcut creator we already built), not hand-placed. Ongoing, only a new product or select needs an occasional one.
People will copy instead of alias
Guardrail: only the curator group adds to the approved view, and the drift monitor watches this folder. The "shortcut, don't copy" step is in Start Here and the one-pager, with the 4-step how-to.
Will a link break
An alias follows the file even if the original is renamed or moved. The only way to break one is to trash the original — so the archive-don't-delete rule keeps them safe.
How do people know it's a link
Aliases show a small arrow badge on the icon, so it's visually obvious an entry points elsewhere.
